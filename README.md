# YFSkateBoard 无线串口通信协议文档

> **适用固件：** YFSkateBoard ESC (STM32F103C8T6, HAL)  
> **通信接口：** UART3（电调板）↔ NRF24L01 ↔ UART1（遥控板）  
> **字节序：** Little-Endian（小端）  
> **浮点数：** IEEE 754 单精度 32-bit  
> **CRC 算法：** CRC-8/MAXIM（poly=0x31，init=0x00）  
> **工具脚本：** `crc_helper.py` — 可生成并验证所有数据包

## 1. 帧结构总览

```
 ┌──────────┬────────────┬───────────┬────────┬──────────────┬──────────┬──────────┐
 │  Start   │ pack_num   │ timestamp │ cmd_id │   Payload    │  End     │  CRC-8   │
 │  0x5B    │ uint32 LE  │ uint32 LE │  u8    │  N bytes     │  0x5D    │ /MAXIM   │
 │  1 byte  │  4 bytes   │  4 bytes  │ 1 byte │ (可变长度)    │  1 byte  │  1 byte  │
 └──────────┴────────────┴───────────┴────────┴──────────────┴──────────┴──────────┘
   byte[0]   byte[1..4]   byte[5..8]  byte[9]  byte[10..N+9]  byte[N+10] byte[N+11]
```

**Header 固定 10 字节 + Payload N 字节 + End 1 字节 + CRC 1 字节 = 总长 N+12 字节**

| 字段 | 偏移 | 大小 | 说明 |
|------|------|------|------|
| `start_flag` | 0 | 1 B | 固定 `0x5B`，帧起始标志 |
| `pack_number` | 1 | 4 B | uint32 LE，滚动包序号（每发一帧自增） |
| `timestamp` | 5 | 4 B | uint32 LE，NTP 同步时间戳（ms） |
| `cmd_id` | 9 | 1 B | 命令 ID，见下表 |
| `payload` | 10 | N B | 命令相关数据（可变长度） |
| `end_flag` | 10+N | 1 B | 固定 `0x5D`，帧结束标志 |
| `CRC-8` | 11+N | 1 B | CRC-8/MAXIM，覆盖 byte[0] 到 byte[10+N]（含 `0x5D`） |

### CRC-8/MAXIM 算法

```c
uint8_t crc8_maxim(const uint8_t *data, uint16_t len) {
    uint8_t crc = 0x00, poly = 0x31;
    while (len--) {
        crc ^= *data++;
        for (uint8_t i = 0; i < 8; i++)
            crc = (crc & 0x80) ? ((crc << 1) ^ poly) : (crc << 1);
    }
    return crc;
}
// 调用：crc8_maxim(buf, end_flag_offset + 1)  — 包含 0x5D
```

```python
# Python 等价实现
def crc8_maxim(data: bytes) -> int:
    crc, poly = 0x00, 0x31
    for b in data:
        crc ^= b
        for _ in range(8):
            crc = ((crc << 1) ^ poly) & 0xFF if (crc & 0x80) else (crc << 1) & 0xFF
    return crc
```

## 3. 控制环切换协议（核心）

### 3.1 控制模式枚举

```c
// Core/Inc/foc.h
typedef enum {
    CTRL_MODE_BLDC     = 0,  // 传统 6-step BLDC（默认/停止）
    CTRL_MODE_TORQUE   = 1,  // 转矩环 / 电流环 — 直接设置 Vq（0~1）
    CTRL_MODE_SPEED    = 2,  // 速度环 — Speed PID → Vq，单位 km/h
    CTRL_MODE_POSITION = 3,  // 位置环 — Position PID → Speed PID → Vq，单位 °
} FOC_ControlMode_t;
```

### 3.2 CMD_FOC_CONTROL (0x40) — 切换控制环

**Payload 结构（9 字节）：**

```c
// Core/Inc/rc_cmd.h
typedef struct __attribute__((packed)) {
    uint8_t mode;           // FOC_ControlMode_t: 0=BLDC 1=TORQUE 2=SPEED 3=POSITION
    float   ref;            // TORQUE: Vq 0~1 | SPEED: km/h | POSITION: 度(°)
    float   current_limit;  // 最大 |Vq| 归一化 0~1，0 = 保持上次设置
} RC_FocControl_t;
```

**Payload 在帧中的位置：** byte[10]=mode, byte[11..14]=ref(float LE), byte[15..18]=current_limit(float LE)

---

#### ▶ 模式 0 — BLDC 停止 / 解除 FOC

```
5B 00 00 00 00 00 00 00 00 40 00 00 00 00 00 00 00 00 00 5D 4F
│  └──pack_num──┘ └─timestamp─┘ │  │  └────ref=0.0────┘ └limit=0.0┘ │  │
0x5B                           0x40 mode=0x00                       0x5D CRC=0x4F
```

| 字段 | 字节 | 值 | 说明 |
|------|------|-----|------|
| start_flag | [0] | `5B` | 帧起始 |
| pack_number | [1..4] | `00 00 00 00` | 包序号 |
| timestamp | [5..8] | `00 00 00 00` | NTP 时间 |
| cmd_id | [9] | `40` | CMD_FOC_CONTROL |
| mode | [10] | `00` | CTRL_MODE_BLDC — 停止 |
| ref | [11..14] | `00 00 00 00` | 0.0 |
| current_limit | [15..18] | `00 00 00 00` | 0.0 |
| end_flag | [19] | `5D` | 帧结束 |
| CRC-8 | [20] | `4F` | ✔ |

---

#### ▶ 模式 1 — 转矩环 / 电流环（TORQUE）

> `ref` = 目标 Vq 归一化值（0~1），`current_limit` = 电流上限

**示例：Vq=0.50，限流=0.8**
```
5B 00 00 00 00 00 00 00 00 40 01 00 00 00 3F CD CC 4C 3F 5D D4
                               ^^                              ^^
                              mode=1(TORQUE)                  CRC
```

| 字段 | 字节 | 值 | IEEE-754 解码 |
|------|------|-----|------|
| mode | [10] | `01` | CTRL_MODE_TORQUE |
| ref | [11..14] | `00 00 00 3F` | **0.50**（Vq = 50%） |
| current_limit | [15..18] | `CD CC 4C 3F` | **0.80** |

**示例：Vq=0.20，限流=0.5（轻柔启动）**
```
5B 00 00 00 00 00 00 00 00 40 01 CD CC 4C 3E 00 00 00 3F 5D 92
```

---

#### ▶ 模式 2 — 速度环（SPEED）

> `ref` = 目标速度（km/h），正数前进，负数倒退

**示例：前进 10 km/h，限流=0.8**
```
5B 00 00 00 00 00 00 00 00 40 02 00 00 20 41 CD CC 4C 3F 5D F4
```

**逐字节解析：**

| 偏移 | 字节 | 值 | 说明 |
|------|------|-----|------|
| [0] | `5B` | — | 帧起始 |
| [1..4] | `00 00 00 00` | — | pack_number = 0 |
| [5..8] | `00 00 00 00` | — | timestamp = 0 |
| [9] | `40` | — | CMD_FOC_CONTROL |
| [10] | `02` | — | **CTRL_MODE_SPEED** |
| [11..14] | `00 00 20 41` | → | **10.0 km/h**（IEEE-754 LE） |
| [15..18] | `CD CC 4C 3F` | → | **0.8**（限流） |
| [19] | `5D` | — | 帧结束 |
| [20] | `F4` | — | CRC-8/MAXIM ✔ |

**示例：前进 25 km/h，满载电流**
```
5B 00 00 00 00 00 00 00 00 40 02 00 00 C8 41 00 00 80 3F 5D 39
```
`C8 41` = 25.0 km/h，`00 00 80 3F` = 1.0（满电流）

**示例：倒退 10 km/h**
```
5B 00 00 00 00 00 00 00 00 40 02 00 00 20 C1 CD CC 4C 3F 5D 18
```
`00 00 20 C1` = -10.0 km/h（IEEE-754 负值）

---

#### ▶ 模式 3 — 位置环（POSITION）

> `ref` = 目标累积角度（°），360° = 1 圈，720° = 2 圈

**示例：目标 720°（2 圈），限流=0.6**
```
5B 00 00 00 00 00 00 00 00 40 03 00 00 34 44 9A 99 19 3F 5D 6E
```

| 字段 | 字节 | 值 |
|------|------|-----|
| mode | [10] | `03` — CTRL_MODE_POSITION |
| ref | [11..14] | `00 00 34 44` → **720.0°** |
| current_limit | [15..18] | `9A 99 19 3F` → **0.6** |

**示例：目标 1080°（3 圈）**
```
5B 00 00 00 00 00 00 00 00 40 03 00 00 87 44 9A 99 19 3F 5D 78
```

### 3.3 模式切换流程

```
遥控板                              电调板 (ESC)
  │                                     │
  │── CMD_FOC_CONTROL(mode=2, ref=10) ──▶│  切换为速度环，目标 10 km/h
  │                                     │  PID 复位，vq→0
  │                                     │  运行 Speed PID
  │◀── CMD_HEARTBEAT(speed=9.8, ...) ───│  速度回报
  │                                     │
  │── CMD_FOC_CONTROL(mode=3, ref=720)─▶│  切换为位置环，目标 720°
  │                                     │  位置 PID 复位
  │◀── CMD_FOC_STATUS(mode=3, pos=720)─│  到达目标
  │                                     │
  │── CMD_FOC_CONTROL(mode=0, ref=0)  ──▶│  停止，回到 BLDC 模式
```

**模式切换时固件行为（Core/Src/foc.c）：**
```c
void FOC_SetMode(FOC_ControlMode_t mode) {
    if (mode != foc.mode) {
        PID_Reset(&foc.pid_speed);      // 速度 PID 复位
        PID_Reset(&foc.pid_position);   // 位置 PID 复位
        foc.vq = 0.0f;                  // 输出清零，防突变
    }
    foc.mode = mode;
}
```

## 4. CMD_FOC_STATUS (0x41) — 状态回报

**双向用途：**
- 遥控板发空 Payload → 请求状态（轮询）
- 电调板主动/应答发送当前状态

**Payload 结构（17 字节）：**

```c
// Core/Inc/rc_cmd.h
typedef struct __attribute__((packed)) {
    uint8_t mode;             // 当前控制模式（FOC_ControlMode_t）
    float   angle_deg;        // 当前电机角度（°）
    float   speed_kmh;        // 当前速度（km/h）
    float   vq;               // 当前 Vq 输出（归一化 0~1）
    float   position_accum;   // 累积位置（°，位置环用）
} RC_FocStatus_t;
```

**示例：请求帧（空 Payload，12 字节）**
```
5B 00 00 00 00 00 00 00 00 41 5D 41
                               ^^  ^^
                            cmd=0x41 end=0x5D CRC=0x41
```

**示例：速度模式回报 — speed=10km/h，angle=120.5°，vq=0.35，pos=4320°（29 字节）**
```
5B 00 00 00 00 00 00 00 00 41 02 00 00 F1 42 00 00 20 41 33 33 B3 3E 00 00 87 45 5D CA
```

| 偏移 | 字节 | 值 | 说明 |
|------|------|-----|------|
| [0] | `5B` | | 帧起始 |
| [1..4] | `00 00 00 00` | | pack_number |
| [5..8] | `00 00 00 00` | | timestamp |
| [9] | `41` | | CMD_FOC_STATUS |
| [10] | `02` | | CTRL_MODE_SPEED |
| [11..14] | `00 00 F1 42` | **120.5°** | angle_deg |
| [15..18] | `00 00 20 41` | **10.0 km/h** | speed_kmh |
| [19..22] | `33 33 B3 3E` | **0.35** | vq |
| [23..26] | `00 00 87 45` | **4320.0°** | position_accum |
| [27] | `5D` | | 帧结束 |
| [28] | `CA` | | CRC-8 ✔ |

**其他模式回报示例：**

| 场景 | 完整字节 |
|------|---------|
| TORQUE — angle=45°，vq=0.50 | `5B 00 00 00 00 00 00 00 00 41 01 00 00 34 42 00 00 00 00 00 00 00 3F 00 00 00 00 5D 5A` |

## 5. CMD_HEARTBEAT (0x30) — 心跳遥测

**方向：** 电调 → 遥控（固定周期发送）  
**Payload（13 字节）：**

```c
typedef struct {
    float   speed;        // 实时速度（km/h）
    uint8_t status;       // 状态位：bit0=低电 bit1=过温 bit2=故障
    float   battery;      // 电池电压（V）
    float   temperature;  // 温度（°C）
} RC_heartbeat_t;
```

| 状态值 | 含义 |
|--------|------|
| `0x00` | 正常运行 |
| `0x01` | 📶 低电量（bit0=1） |
| `0x02` | 🌡 过温（bit1=1） |
| `0x03` | 低电 + 过温 |

**示例：正常运行 speed=10km/h，bat=36.5V，temp=25°C（25 字节）**
```
5B 00 00 00 00 00 00 00 00 30 00 00 20 41 00 00 00 12 42 00 00 C8 41 5D EB
```

| 偏移 | 字节 | 值 | 说明 |
|------|------|-----|------|
| [10..13] | `00 00 20 41` | **10.0 km/h** | speed |
| [14] | `00` | 正常 | status |
| [15..18] | `00 00 12 42` | **36.5 V** | battery |
| [19..22] | `00 00 C8 41` | **25.0 °C** | temperature |
| [23] | `5D` | | 帧结束 |
| [24] | `EB` | | CRC ✔ |

**其他场景：**

| 场景 | 完整字节 |
|------|---------|
| 低电 bat=32V，速度 5km/h | `5B 00 00 00 00 00 00 00 00 30 00 00 A0 40 01 00 00 00 42 00 00 F0 41 5D B2` |
| 过温 temp=85°C，停止 | `5B 00 00 00 00 00 00 00 00 30 00 00 00 00 02 00 00 0C 42 00 00 AA 42 5D A7` |
| 满电静止 bat=42V | `5B 00 00 00 00 00 00 00 00 30 00 00 00 00 00 00 00 28 42 00 00 B0 41 5D BC` |

## 7. CMD_TIME_SYNCHRONOUS (0x13) — NTP 时钟同步

**用途：** 计算遥控板和电调的时钟偏差，补偿传输延迟  
**Payload（16 字节）：**

```c
// t1 由遥控板发出时打，t2 电调收到时打，t3 电调回复时打，t4 遥控板收到时打
typedef struct __attribute__((packed)) {
    uint32_t t1;   // 请求发出时刻（遥控板 ms）
    uint32_t t2;   // 请求收到时刻（电调 ms）
    uint32_t t3;   // 响应发出时刻（电调 ms）
    uint32_t t4;   // 响应收到时刻（遥控板 ms）
} RC_TimeSync_t;
// 时钟偏差 offset = ((t2-t1)+(t3-t4)) / 2
// 单程延迟 delay  = ((t4-t1)-(t3-t2)) / 2
```

**示例：t1=1000 t2=1001 t3=1002 t4=1003（28 字节）**
```
5B 00 00 00 00 00 00 00 00 13 E8 03 00 00 E9 03 00 00 EA 03 00 00 EB 03 00 00 5D AB
```

| 偏移 | 字节 | 值 |
|------|------|-----|
| [10..13] | `E8 03 00 00` | **1000** ms |
| [14..17] | `E9 03 00 00` | **1001** ms |
| [18..21] | `EA 03 00 00` | **1002** ms |
| [22..25] | `EB 03 00 00` | **1003** ms |
| [27] | `AB` | CRC ✔ |

## 9. 各控制环对比总结

| 控制环 | mode 值 | ref 含义 | 单位 | 典型场景 |
|--------|---------|----------|------|---------|
| BLDC（停止） | `0x00` | 无效 | — | 解除控制，回到 6-step 模式 |
| **转矩环**（电流环） | `0x01` | Vq 占比 | 0~1 | 爬坡时精确扭矩控制 |
| **速度环** | `0x02` | 目标速度 | km/h | 日常骑行巡速 |
| **位置环** | `0x03` | 目标角度 | 度(°) | 精确定位/驻车 |

**切换建议：**
- 速度环 → 停止：先发 `mode=2, ref=0.0`，待速度降到 0 再发 `mode=0`
- 位置环 → 速度环：直接切换，PID 自动复位
- `current_limit` 字段设为 `0.0` 可保持上次配置，无需每次传送

## 11. FOC 预定义常量速查（firmware 侧）

以下常量在切换控制环时需要提前在固件 / 上位机两侧约定一致。

### 11.1 控制模式（`rc_cmd.h` payload byte[0]）

```c
#define CTRL_MODE_BLDC      0x00   // 传统 6-step，停止 FOC
#define CTRL_MODE_TORQUE    0x01   // 转矩环 / 电流环
#define CTRL_MODE_SPEED     0x02   // 速度环
#define CTRL_MODE_POSITION  0x03   // 位置环
```

### 11.2 CMD_FOC_CONTROL payload 字段范围

| 字段 | 类型 | 合法范围 | 说明 |
|------|------|---------|------|
| `mode` | `uint8_t` | 0~3 | 见上表 |
| `ref` | `float` | TORQUE: 0.0~1.0 / SPEED: -30~30 km/h / POSITION: 任意 °  | 目标值 |
| `current_limit` | `float` | 0.0~1.0，0.0=保持 | 最大 \|Vq\| |

### 11.3 固件模式切换时自动复位行为

```c
void FOC_SetMode(FOC_ControlMode_t mode) {
    if (mode != foc.mode) {
        PID_Reset(&foc.pid_speed);     // 速度积分清零
        PID_Reset(&foc.pid_position);  // 位置积分清零
        foc.vq = 0.0f;                 // 输出立即归零，防止突变
    }
    foc.mode = mode;
}
```

> **注意：** 切换模式时 `vq` 会瞬间置 0，实际输出会有约 1 个 PWM 周期（33 µs @ 30 kHz）的中断，低速切换无感，高速切换会有轻微抖动，建议在速度低于 3 km/h 时切换。

## 12. 实操示例集——速度环 / 转矩环 / 位置环

> 所有数据包均通过 `crc_helper.py` 实际运行生成，CRC 已验证。  
> **帧格式：** `[5B][pack_num 4B][timestamp 4B][cmd_id][payload][5D][CRC]`  
> **payload byte[0]** = mode，**byte[1..4]** = ref（float LE），**byte[5..8]** = current_limit（float LE）

### 12.3 转矩环 / 电流环（TORQUE，mode=0x01）— 小中大最大

> `ref` = 目标 Vq 归一化（0~1），`current_limit` = 保护上限（≥ ref）  
> Vq 越大 = 电流越大 = 扭矩越大（无速度反馈，开环）

#### 例 1 — 小档（小扭矩，Vq=0.15，限流=0.3）
```
5B 00 00 00 00 00 00 00 00 40 01 9A 99 19 3E 9A 99 99 3E 5D E5
```
| 偏移 | 字节 | 值 | 说明 |
|------|------|-----|------|
| [10] | `01` | CTRL_MODE_TORQUE | |
| [11..14] | `9A 99 19 3E` | **0.15** Vq | 轻微爬坡助力 |
| [15..18] | `9A 99 99 3E` | **0.30** 限流 | |
| [20] | `E5` | CRC ✔ | |

#### 例 2 — 中档（中等扭矩，Vq=0.40，限流=0.6）
```
5B 00 00 00 00 00 00 00 00 40 01 CD CC CC 3E 9A 99 19 3F 5D B6
```
`CD CC CC 3E` = 0.40，`9A 99 19 3F` = 0.60，CRC=`B6` ✔

#### 例 3 — 大档（强扭矩，Vq=0.70，限流=0.9）
```
5B 00 00 00 00 00 00 00 00 40 01 33 33 33 3F 66 66 66 3F 5D 44
```
`33 33 33 3F` = 0.70，`66 66 66 3F` = 0.90，CRC=`44` ✔

#### 例 4 — 最大（全力矩，Vq=1.00，限流=1.0）
```
5B 00 00 00 00 00 00 00 00 40 01 00 00 80 3F 00 00 80 3F 5D B7
```
`00 00 80 3F` = 1.00（×2），CRC=`B7` ✔

> ⚠️ 最大档仅短时使用，长时间全力矩会导致电调过热。

### 12.6 完整操作序列示例

#### 场景 A — 从停止启动到 10 km/h 巡速

```
① 解除停止（BLDC mode）
   → 5B 00 00 00 00 00 00 00 00 40 00 00 00 00 00 00 00 00 00 5D 4F

   → 5B 00 00 00 00 00 00 00 00 40 02 00 00 00 40 CD CC 4C 3F 5D 8C

③ 加速到 10 km/h
   → 5B 00 00 00 00 00 00 00 00 40 02 00 00 20 41 CD CC 4C 3F 5D F4

④ 查询当前状态
   → 5B 00 00 00 00 00 00 00 00 41 5D 41
   ← ESC 回报 speed=10km/h

⑤ 减速刹停
   → 5B 00 00 00 00 00 00 00 00 40 02 00 00 00 00 00 CD CC 4C 3F 5D xx

   → 5B 00 00 00 00 00 00 00 00 40 00 00 00 00 00 00 00 00 00 5D 4F
```
